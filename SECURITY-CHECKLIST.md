# چک‌لیست امنیتی هنرستان هادی

## تنظیمات محیطی (.env)

قبل از استقرار production، موارد زیر را بررسی کنید:

```bash
# الزامی
JWT_SECRET=<یک رمز قوی و منحصر به فرد>
DATABASE_URL=<آدرس دیتابیس>
ADMIN_SECRET_PATH=<مسیر مخفی admin>

# اختیاری اما توصیه شده
NODE_ENV=production
```

## بررسی‌های قبل از استقرار

### ۱. امنیت عمومی
- [ ] `JWT_SECRET` تنظیم شده و قوی است (حداقل 32 کاراکتر)
- [ ] `ADMIN_SECRET_PATH` به مسیر غیرقابل حدس تغییر کرده
- [ ] `NODE_ENV=production` تنظیم شده
- [ ] رمز عبور پیش‌فرض admin تغییر کرده

### ۲. دیتابیس
- [ ] فایل `.db` در معرض عموم نیست
- [ ] بکاپ منظم تنظیم شده
- [ ] دسترسی مستقیم به دیتابیس محدود است

### ۳. آپلود فایل
- [ ] پوشه `public/uploads` محدودیت حجم دارد
- [ ] فقط فایل‌های تصویری مجاز هستند
- [ ] نام فایل‌ها sanitize می‌شوند

### ۴. شبکه
- [ ] HTTPS فعال است
- [ ] Security headers تنظیم شده
- [ ] CSP تنظیم شده

### ۵. لاگ‌ها
- [ ] لاگ‌های امنیتی بررسی می‌شوند
- [ ] تلاش‌های لاگین ناموفق مانیتور می‌شوند
- [ ] لاگ‌ها شامل IP و زمان هستند

## مسیرهای مخفی

| مسیر | توضیح |
|------|-------|
| `/hadi-panel-x7k9` | پنل مدیریت (قابل تغییر) |
| `/api/auth` | لاگین |
| `/api/auth/logout` | خروج |
| `/api/auth/change-password` | تغییر رمز |

## API‌های امنیتی

| endpoint | مethod | نیاز به auth | توضیح |
|----------|--------|--------------|-------|
| `/api/admin/security-logs` | GET | ✅ | لاگ‌های امنیتی |
| `/api/admin/login-attempts` | GET | ✅ | تلاش‌های لاگین |
| `/api/admin/backup` | POST | ✅ | ایجاد بکاپ |
| `/api/admin/backup` | GET | ✅ | لیست بکاپ‌ها |

## بکاپ و بازیابی

### ایجاد بکاپ
```bash
./scripts/backup.sh
```

### بازیابی بکاپ
```bash
./scripts/restore.sh <timestamp>
```

### بکاپ خودکار (cron)
```bash
# هر روز ساعت 2 بامداد
0 2 * * * cd /path/to/project && ./scripts/backup.sh
```
